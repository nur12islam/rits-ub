Commands & Usage

Learn how to use all the powerful features of RITS Userbot. All commands are triggered by the configured prefix (default is ! or .).

🛡️ Administration

👤 User Management

!ban [reply/id] - Ban a user from the chat
!unban [reply/id] - Unban a user
!mute [reply/id] - Mute a user
!unmute [reply/id] - Unmute a user
!kick [reply/id] - Kick a user
!promote [reply/id] - Promote a user to admin
!demote [reply/id] - Demote an admin
!block [reply/id] - Block a user (PM)

💬 Chat Management

!pin [reply] - Pin a message
!gpic [reply to image] - Change group picture
!purge [reply] - Delete messages from replied
!tagall [text] - Tag all members in chat

🛠️ Utilities & Media

!ytdl [url] - Download YouTube video/audio
!ytinfo [url] - Get YouTube video info
!ocr [reply to image] - Extract text from image
!weather [city] - Get current weather
!calc [expression] - Calculate math expression
!speedtest - Run server speedtest

👨‍💻 Developer & System

!eval [code] - Execute TS/JS code
!exec [command] - Execute shell command
!install [reply to .ts] - Install a new plugin
!logs - Get bot error logs
!restart - Restart the bot process
!setvar [key] [val] - Set environment variable

📩 PM Permit (Security)

PM Permit automatically blocks unwanted direct messages from strangers unless they are approved by you.

!pmguard [on/off] - Enable/Disable PM Permit
!allow [reply/id] - Approve a user to PM you
!nopm [reply/id] - Disallow a user
!setpmmsg [text] - Set custom warning message

Plugin Development

RITS Userbot is highly extensible. You can write your own plugins and load them dynamically.

How to add a new plugin

Method 1: Using the !install command

Simply write your plugin in a .ts or .js file, send it in any chat, reply to the file and type !install. The bot will automatically save it and reload without restarting!

Method 2: Adding directly to GitHub (Source Code)

1. Navigate to src/bot/plugins/ in your repository.
2. Create a new file (e.g. myplugin.ts).
3. Add your code using the standard plugin structure (see below).
4. Important: Open src/bot/pluginManager.ts and add your new plugin to the static import list at the top to ensure it gets bundled by Vite during deployment! 

import { NewMessageEvent } from "telegram/events/index.js";

export default [{
    name: "MyPlugin",
    description: "My custom plugin",
    command: "mycmd",
    category: "Custom",
    handler: async (event: NewMessageEvent) => {
        await event.message.edit({ text: "Hello World!" });
    }
}];

Deployment Guide

Deploy RITS Userbot easily on popular free platforms like Render.

Deploying on Render.com

  1. Push your code to a new GitHub Repository.
  2. Go to Render and create a new account if you don't have one.
  3. Click New > Web Service.
  4. Connect your GitHub account and select your userbot repository.
  5. Fill in the deployment details:
    • Name: rits-userbot
    • Region: Singapore (Asia) - Crucial for better latency
    • Environment: Node
    • Build Command: npm install && npm run build
    • Start Command: npm start
  6. Environment Variables: Scroll down to "Environment Variables" and click "Add Environment Variable". Add the following:
    • API_ID: (Get from my.telegram.org)
    • API_HASH: (Get from my.telegram.org)
    • SESSION: (Paste your generated Session String here)
    • BOT_TOKEN: (Optional: from @BotFather for Assistant Bot)
  7. Click Create Web Service. Your bot will deploy and be active within a few minutes!

Important Note about Render Free Tier

Render free instances spin down after 15 minutes of inactivity. Since Web Services require incoming HTTP requests to stay awake, and Telegram MTProto doesn't send HTTP pings to your server, you need to use a free pinging service (like UptimeRobot) pointing to your web service URL to keep it running 24/7. When setting up UptimeRobot, try to select an Asia region for the monitoring node to reduce latency.
